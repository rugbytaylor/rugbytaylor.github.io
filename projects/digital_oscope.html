Embedded Digital Oscilloscope

Objective

Design a fully embedded, battery-powered digital oscilloscope capable of measuring -5V to +5V signals using an ESP32, with real-time waveform display, on-screen cursor tools, waveform freeze/screenshot capture, and accurate voltage readout. Beyond the hardware, a major goal of this project was to build out a complete embedded software pipeline in C, including version control and CI/CD automation, with a planned path to OTA firmware updates. This was a solo project completed over two months, started as a personal extension of an ESP32 game console I built in a class, done entirely on my own time to learn more about embedded systems.

Key Requirements

  • -5V to +5V input range, single channel
  • 10-20 kHz signal measurement target
  • <10% measurement error target
  • Fully embedded and battery powered, with on-board charging
  • Real-time waveform display with cursor, freeze, and screenshot capture

System Architecture

The analog front-end is a simple op-amp scaling circuit built around an MCP6002 op amp, which scales the -5V to +5V input range down into the 0-3.3V range the ESP32's ADC can accept. This was designed and verified in LTspice before moving to hardware, as shown below.

The rest of the system is built around the ESP32, which samples the conditioned signal, renders the waveform in real time on a small LED screen, and reads input from a joystick and buttons for navigating the display, freezing the waveform, and capturing screenshots. An SD card slot on the screen module doubles as both a calibration tool during development and a way to store captured waveform data. Power comes from an onboard 3.3V lithium battery with charging circuitry, making the whole board self-contained and portable.

On the software side, the entire application was written in C. I also built out a full development pipeline for the project using GitHub Actions for CI/CD, with a planned future addition of OTA (over-the-air) firmware updates so the oscilloscope can be updated without a physical connection.

Design Challenges and Iterations

The hardest problem was getting the real-time waveform display to render accurately without aliasing. Balancing ADC sample rate against how quickly the screen could redraw was a constant tradeoff: sampling too slowly caused aliasing on faster signals, while refreshing too aggressively made the waveform difficult to read. Finding a sample rate and display refresh combination that kept 10-20 kHz signals both accurate and visually legible took a lot of iteration.

The other major challenge was calibration. Because the analog front-end introduced its own small nonlinearities, I built a custom lookup table (LUT) to map raw ADC readings back to real input voltage. To calibrate it, I fed a known waveform from a benchtop waveform generator into the circuit, logged the ESP32's sampled voltage and timing data to one SD card, and simultaneously logged the waveform generator's actual output to a second SD card. I then wrote a MATLAB script to align and compare the two datasets and generate the LUT relating raw ADC readings to true input voltage. Getting the on-screen cursor to correctly report LUT-corrected values (rather than raw ADC counts) took additional tuning, but wasn't conceptually difficult, just time-consuming to get right.

The board itself went through three PCB iterations. The first revision had several ESP32 pins wired incorrectly, and later revisions mostly corrected footprint sizing issues and refined the op-amp scaling circuit until it performed as designed in simulation.

Validation and Results

I validated total system accuracy the same way I calibrated it: feeding a known waveform into the circuit, capturing the ESP32's measured values on one SD card and the waveform generator's actual output on another, then using a second MATLAB script to directly compare the two datasets and calculate measurement error.

The final system achieved <2% measurement error, well beyond the original target of <10% error. Most of that improvement came from the custom LUT correcting for nonlinearities in the analog front-end that the raw ADC readings alone could not account for.

Results

The final system met and exceeded all key requirements: a fully embedded, battery-powered oscilloscope measuring -5V to +5V signals up to 10-20 kHz, with real-time waveform display, cursor and freeze/screenshot functionality, and better than 2% total measurement error. The project is currently built and validated on a breadboard; an enclosure was not built as part of this iteration.
